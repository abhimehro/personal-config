- Testing bash shell scripts with complex paths and sub-commands: Mocks can be established dynamically using `eval` or by exporting mocked bash functions into the test file. The path manipulation using `$MOCK_BIN` works perfectly for commands called natively by name without path bindings.
- When creating tests for a complex orchestrator script that sources other libraries, make sure the sourced paths map correctly to the repo root before altering the path inside test mocks.
