## Local Git / Jujitsu (jj) note

Merges and closes were performed with **`gh pr merge` / `gh pr close`** using `GH_TOKEN` (user account). **Direct `git push` to PR branches** in this environment used **`cursor[bot]`** credentials and returned **403**; for jj-first workflows, prefer **`gh`** for branch updates or configure `credential.helper` / `gh auth setup-git` so HTTPS pushes use the **same PAT as `GH_TOKEN`**, not a secondary bot token from `~/.config/gh/hosts.yml`.
